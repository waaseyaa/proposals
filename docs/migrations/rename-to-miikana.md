# Rename Plan — `waaseyaa-proposals` → `miikana`

**Status:** proposed, not executed. No file modifications in this pass.
**Target name:** Miikana (product). Deployment host: `miikana.waaseyaa.org`.
**Framework:** stays `waaseyaa/*`. This is a *product* rename, not a framework rename.

## Scope summary

The code does not carry the old product name as a PHP namespace — root is `App\`.
The rename surface is therefore **composer metadata + documentation + UI strings + corpus/seed command names**. It is a one-day job if the vendor-prefix decision is resolved first.

### Decision required (blocking)

**Who owns this package, `northops/` or `waaseyaa/`?**
- `northops/miikana` — NorthOps keeps authorship, Miikana is NorthOps' product
- `waaseyaa/miikana` — Miikana is treated as a first-party Waaseyaa product (matches `miikana.waaseyaa.org`)

Recommendation: `waaseyaa/miikana`. The product ships on `*.waaseyaa.org`, the framework sibling is `waaseyaa/framework`, and `waaseyaa/bimaaji` is already a first-party dep. Keeping vendor alignment matches the hosted-tier story. NorthOps-as-tenant remains a data concept (`Organization`), not a package-vendor concept.

Not found in the audit and not claimed. Treat as: **not published, no Packagist retraction needed**. Verify before executing by searching `composer show waaseyaa/miikana` and `composer show northops/waaseyaa-proposals` against the live registry during execution.

### UI-string rename rules

- **CLI command description:** update in the same PR.

### CLI command name

The command is registered as `proposals:seed-northops` (`src/Command/SeedNorthOpsCommand.php` `#[AsCommand]` attribute). Rename path:
- `proposals:seed-northops` → `miikana:seed-northops` — same scope, new prefix
- OR `northops:seed` — makes clearer this is the *NorthOps tenant's* seeder, not a product-wide command. This matches §northops-seed-extraction direction. **Recommended.**

## 8. Proposed order of operations

Execute in this order in a single branch. Each step is a separate commit so rollback is mechanical.

1. **Pre-flight (read-only)** — confirm the deltas in this doc are still accurate: `grep -rn 'waaseyaa-proposals\|WaaseyaaProposals\|Waaseyaa Proposals' --include='*.php' --include='*.json' --include='*.md'`. If anything new appears, stop and update this doc.
2. **Resolve vendor-prefix decision** (Russell). Recommended: `waaseyaa/miikana`.
3. **composer.json rename** — `name`, `description`. Run `composer update --lock` (locked, no new deps). Commit.
4. **CLI command rename** — `proposals:seed-northops` → `miikana:seed-northops` (or `northops:seed`, per §6). Single-file change. Commit.
5. **User-visible strings** — HTML titles, dashboard copy, DuckDuckGo User-Agent, CLI description, seeded `conversation_summary`. Commit.
6. **`CLAUDE.md` + `README.md`** — product rename + positioning. Commit.
7. **Smoke test** — `bin/waaseyaa serve` + browse `/`, `/submissions`, `/cohorts`. Confirm title bar and dashboard now read "Miikana." No test suite to run (empty `tests/`).
8. **Tag** — `v0.1-miikana-rename` for rollback anchor.

Do **not** execute any of the above until §9 is signed off.

## 9. Breaking-change surfaces (flag for human decision)

| Surface | Risk | Decision |
|---|---|---|
| `composer.json` `name` change | If any sibling repo consumes this via path repository or Packagist, its `composer.json`/`composer.lock` breaks | Confirm no consumers exist (none found at audit time, but Russell must confirm local `composer.local.json` siblings in `~/dev/*` aren't referencing it) |
| CLI command rename | Any automation calling `bin/waaseyaa proposals:seed-northops` breaks | Grep external scripts + shell history; provide short grace window by temporarily registering both names? Recommend **hard cutover**: 2 commits on main, no one else runs this command yet |
| `WaaseyaaProposalsResearchBot` User-Agent | External services that whitelist by User-Agent break (DuckDuckGo doesn't; no other consumers) | None known. Rename freely. |
| Dashboard HTML/title | Bookmarks and screenshots in docs drift | Cosmetic; acceptable |

### Not a breaking change

- PHP namespace: no change (stays `App\`)
- Route paths: no change
- Env var names: no change
- Database schema: no change
- Entity type IDs (`proposal_pipeline` etc.): **intentionally not renamed in this pass** — they are data-model concepts that outlive product names. If a rename is ever wanted, that is a Tier-3 data migration and does not belong here.

## 10. Out of scope (deferred)

- **GitHub repo rename.** Can be done after the code-level rename ships. GitHub auto-redirects.
- **Local filesystem directory rename** (`~/dev/waaseyaa-proposals/`). Zero benefit; high risk to in-flight workspace state.
- **Entity type rename** (`proposal_*` → `miikana_*`). Do not do this. Entity IDs are data-model nouns; "proposal" is accurate domain language regardless of product branding.
- **Framework rename.** `waaseyaa/*` packages are not affected.
